# 开发会话记录

本目录记录开发过程中的详细会话日志，包括与AI的交互、开发思路和决策过程，形成可追溯的开发历史。

## 目的

通过记录开发会话，我们可以：

1. **知识积累** - 记录开发过程中的思考和决策
2. **上下文保持** - 为后续开发提供必要的背景信息
3. **经验分享** - 帮助团队成员了解开发过程
4. **AI协作优化** - 为AI提供历史交互记录，改进协作效果

## 会话记录结构

每个会话记录应包含以下部分：

```markdown
# [日期]_[时间]_[会话主题].md

## 目标
描述本次会话的目标和预期结果

## 思考过程
记录开发过程中的思考、分析和决策过程

## 决策与实现
记录做出的决策和具体的实现步骤

## 结果与后续
记录会话的结果和后续需要跟进的事项

## 关联文件
- [相关文件1](链接)
- [相关文件2](链接)
```

## 使用方法

### 对于AI助手
1. 在开始新的开发任务前，查阅相关的会话记录
2. 了解之前的开发思路和决策过程
3. 基于历史记录提供一致的协助
4. 在会话结束后，帮助整理和记录会话内容

### 对于开发者
1. 完成一个相对独立的开发任务后，创建会话记录
2. 记录开发过程中的思考、决策和实现
3. 确保记录清晰、完整，便于后续查阅
4. 定期整理和归档会话记录

## 文件命名约定

会话记录文件应采用以下命名格式：
`YYYY-MM-DD_HH-MM_[会话主题].md`

其中：
- `YYYY-MM-DD` 为会话日期
- `HH-MM` 为会话开始时间
- `[会话主题]` 为简短的会话描述，使用下划线分隔单词

## 维护指南

- 定期创建会话记录，记录重要的开发过程
- 确保记录的结构化和可读性
- 避免记录过于琐碎的细节，关注重要的决策和思路
- 定期审查和整理会话记录，移除过时或无关的内容
